## Overview

On this page, we introduce how handlers can be attached to various Kintone events. The different types of Kintone events are listed in the side menu.

## Register Event Handlers - `kintone.events.on(type, handler)`

Registers an event handler.

### Available Pages

- [Pages Where Kintone System-wide Customizations Are Applied](/docs/kintone/js-api/overview/kintone-js-api-specifications#pages-system-wide-customization)
- Plug-in settings page

### Function

```
kintone.events.on(type, handler)
```

### Parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| type | String or Array of Strings | Yes | The event type or array of event types to which the event handler will bind. |
| handler | Function(Object) | Yes | The handler that will run when the event is triggered. All event objects have an event type in their `type` property. By returning a `Promise` object in the event handler, the next operations can be processed after waiting for asynchronous processes to finish. |

#### Events that support returning a `Promise` object

**Record List Events**
- [Onload Event](/docs/kintone/js-api/events/record-list-events/onload-event#onload-event-desktop)
- [Inline Edit Event](/docs/kintone/js-api/events/record-list-events/inline-edit-event)
- [Save Event](/docs/kintone/js-api/events/record-list-events/save-event)
- [Save Success Event](/docs/kintone/js-api/events/record-list-events/save-success-event)
- [Delete Event](/docs/kintone/js-api/events/record-list-events/delete-event)

**Record Create Events**

- [Save Event](/docs/kintone/js-api/events/record-create-events/save-event)
- [Save Success Event](/docs/kintone/js-api/events/record-create-events/save-success-event)

<Note>
  A `Promise` object is an object that holds a "then" method. See [MDN Web Docs](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise) for more details.
</Note>

### Returns

Nothing

### Sample

```javascript
var handler = function(event) {
  console.log(event);
};

kintone.events.on('app.record.index.show', handler);

kintone.events.on(['app.record.detail.show', 'app.record.edit.show'], handler);
```

## Remove Event Handlers - `kintone.events.off(type, handler)`

Removes event handlers from the specified event type(s).

### Available Pages

- [Pages Where Kintone System-wide Customizations Are Applied](/docs/kintone/js-api/overview/kintone-js-api-specifications#pages-system-wide-customization)
- Plug-in settings page

### Function

```
kintone.events.off(type, handler)
```

### Parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| type | String or Array of Strings | Optional | The event type(s) to which the event handler is bound. |
| handler | Function(Object) | Optional | The handler to remove from the specified event type(s). If omitted, all handlers are removed from the specified event type(s). If both `type` and `handler` are omitted, all handlers from all event types are removed. |

### Returns

`true` if any event handlers were removed.  
`false` if no event handlers were removed.

### Sample

```javascript
var handler = function(event) {
  console.log(event);
};

kintone.events.on('app.record.index.show', handler);

// Option 1: Remove the specified handler from the specified event type
kintone.events.off('app.record.index.show', handler);

// Option 2: Remove all handlers from the specified event type
kintone.events.off('app.record.index.show');

// Option 3: Remove all handlers from all event types
kintone.events.off();
```

## Limitations

- If there are JavaScript files for both apps and plug-ins in the same app, files are loaded in the following order:
  1. JavaScript files for apps
  2. JavaScript files for plug-ins
- Event handlers cannot be overwritten. The existing event handler must be deleted before adding a new one.
- If there are multiple categories with the same name within the same App, an unintended category may be returned at the following events when returning an object containing the `category` property. To avoid this, return the object after deleting the `category` property.
